# 2026-06-18 响应式布局实施计划

## 目标

让 lc-agent 网页同时适配电脑和手机浏览器：

- 电脑端保持现有三栏工作台体验。
- 手机端只展示主聊天区，会话列表改为左侧抽屉，工具/状态面板改为右侧抽屉。
- 顶部栏在窄屏下收纳低优先级按钮，保留会话入口、Agent 选择、新对话、工具入口和连接状态。
- 聊天消息、工具调用结果、代码块和输入框不能产生横向页面滚动。

## TDD 验收方式

当前前端没有 Vitest/Playwright 测试依赖。为了不引入额外网络依赖，本次先添加一个轻量的 Node 契约测试：

- 新增 `frontend/scripts/check-responsive-layout.mjs`。
- 新增 npm 脚本 `test:responsive`。
- 测试读取 Vue/CSS 源码，断言响应式布局必须具备的结构、类名、断点和移动端可访问按钮。

红灯预期：

- 测试先失败，因为当前没有移动端抽屉状态、移动端按钮、响应式断点和聊天窄屏规则。

绿灯预期：

- 实现后 `npm run test:responsive` 通过。
- `npm run build` 通过。
- 重启本地服务后，用浏览器验证桌面宽度和手机宽度都能正常操作。

## 实施步骤

1. 添加响应式契约测试。
   - 覆盖 `App.vue` 的移动端左右抽屉状态、遮罩和打开事件。
   - 覆盖 `AppHeader.vue` 的移动端按钮、中文 `aria-label` 和可压缩 Agent 选择器。
   - 覆盖 `LeftSidebar.vue` 与 `RightPanel.vue` 的移动端宽度断点。
   - 覆盖 `ChatView.vue` 与 `ChatInput.vue` 的窄屏消息和输入框规则。
   - 先运行测试确认失败。

2. 实现 App 级响应式框架。
   - 在 `App.vue` 增加 `mobileLeftOpen`、`mobileRightOpen` 状态。
   - 左右抽屉互斥打开，点击遮罩关闭。
   - 切换会话、新建会话后关闭移动端抽屉。
   - 手机端左栏打开时强制展示完整会话列表，不沿用桌面折叠态。

3. 改造顶部栏。
   - 增加“打开会话列表”和“打开工具和状态面板”两个移动端按钮。
   - 移除 Agent 选择器的内联固定宽度，改为 CSS 类控制。
   - 在移动端隐藏编辑 Agent、新 Agent 等低频按钮，避免顶部挤爆。

4. 改造左右面板和聊天区 CSS。
   - `900px` 以下将左右面板变为固定定位抽屉。
   - `520px` 以下进一步压缩聊天区和输入框间距。
   - 消息气泡、工具结果、代码块限制在视口内，内部横向滚动而不是整页横向滚动。

5. 验证和重启。
   - 运行 `npm run test:responsive`。
   - 运行 `npm run build`。
   - 使用项目的 `restart-bfzs` 脚本重启本地页面。
   - 使用浏览器分别验证桌面和手机宽度。

## 风险和取舍

- 这次不引入新的测试框架，避免为了响应式改造额外下载依赖。
- 契约测试不是视觉回归测试，所以最后仍然需要浏览器实际验证。
- 右侧工具/状态面板手机端默认隐藏，会多一次点击，但换来聊天主路径更顺。
